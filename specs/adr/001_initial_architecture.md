# ADR 001: 初期アーキテクチャの選定

## ステータス
承認済み

## コンテキスト
Streamer Link Portal は、低コストかつ高スケーラビリティを実現する必要がある。また、管理画面（管理者用）と公開ページ（ファン用）で求められる特性が異なる。

## 意思決定
以下の技術スタックを採用する。

1. **フレームワーク**: Next.js (App Router)
   - SSR による動的な管理画面と、ISR による静的な公開ページの出し分けが可能。
2. **バックエンド**: Supabase (PostgreSQL + Auth)
   - サーバーレスで完結し、RLS によるセキュアなデータアクセスが可能。
3. **ホスティング**: Vercel
   - Next.js との親和性が高く、Edge 配信が容易。

## 理由
- 開発スピードの最大化。
- 無料枠（Free Tier）内での運用が容易。
- 完全サーバーレス構成により、サーバーのメンテナンスが不要。

## 影響
- TypeScript による型安全な開発が可能。
- Supabase のクォータ制限（APIリクエスト数など）を意識した設計が求められる。
